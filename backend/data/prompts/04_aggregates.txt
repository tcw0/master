## Instructions

### 1. Design Each Aggregate
For each aggregate identified in the bounded contexts:

**Basic Information:**
- **name**: Aggregate name in PascalCase (e.g., 'Booking', 'Room')
- **description**: Purpose of this aggregate
- **bounded_context**: Which context it belongs to
- **root_entity**: Name of the aggregate root entity
- **identity**: The identity attribute (name, type, description)

**Root Attributes:**
List attributes of the aggregate root (excluding identity):
- **name**: Attribute name in camelCase
- **type**: Data type (string, DateTime, or custom type name)
- **is_required**: Whether required
- **description**: Business meaning (optional)

**Contained Elements:**

*Entities* (child entities with identity, lifecycle controlled by root):
- **name**: Entity name
- **description**: What it represents
- **identity_attribute**: What uniquely identifies it
- **attributes**: List of attributes
- **value_objects**: VOs used by this entity
- **relationship_to_root**: How it relates to the root

*Value Objects* (immutable, identity-less):
- **name**: VO name (e.g., 'TimeSlot', 'EmailAddress')
- **description**: What it represents
- **attributes**: Attributes that define it
- **validation_rules**: Rules for validity
- **is_reusable**: Whether used across aggregates

*Enums* (fixed set of values):
- **name**: Enum name (e.g., 'BookingStatus')
- **description**: What it represents
- **values**: List of values in UPPER_SNAKE_CASE

**Behavior:**

*Invariants* (business rules to protect):
- **name**: Short invariant name
- **description**: The business rule in plain language
- **enforcement**: How it's enforced

*Commands* (operations the aggregate handles):
- **name**: Command name (e.g., 'createBooking', 'cancel')
- **description**: What it does
- **parameters**: Input parameters
- **preconditions**: What must be true before
- **postconditions**: State after success
- **events_raised**: Events produced
- **exceptions**: Errors that can occur

*Domain Events* (events this aggregate publishes):
- List of event names

**Design Evaluation:**
- **consistency_boundary**: What's consistent within this aggregate
- **size_evaluation**: 'appropriate', 'possibly_too_large', 'possibly_too_small'
- **size_notes**: Recommendations if concerns exist
- **references_aggregates**: Other aggregates referenced by ID

### 2. Shared Value Objects
List VOs used across multiple aggregates

### 3. Cross-Aggregate Interactions
Document event-based interactions between aggregates:
- **source_aggregate**: Publisher
- **event**: Event name
- **target_aggregate**: Reactor
- **reaction**: What happens
- **consistency**: 'eventual' or 'saga'

### 4. Design Concerns
List any aggregates that may need restructuring

### 5. Missing Invariants
Note business rules not yet covered

Ensure aggregates are small (for concurrency), focused on a single consistency boundary, and protect clear business invariants.
